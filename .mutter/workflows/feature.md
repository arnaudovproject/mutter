# Workflow: feature

1. Read `.mutter/architecture/overview.md` and relevant index shards.
2. `/mutter:brainstore` or refresh domain notes if the feature is new ground.
3. `/mutter:plan` with scope; list affected files and risks.
4. `/mutter:task` split into verifiable steps under `.mutter/tasks/current/`.
5. `/mutter:safe-edit` per step; run tests as defined in the task.
6. `/mutter:review-diff`; update `architecture/decisions.md` if boundaries changed.
