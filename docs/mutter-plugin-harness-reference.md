## Claude Code (`mutter-claude`)

Docs: [Create plugins](https://code.claude.com/docs/en/plugins), [Plugins reference](https://code.claude.com/en/plugins-reference).

- Manifest: `mutter-claude/.claude-plugin/plugin.json` — `name: mutter` ⇒ skills are **`/mutter:<skill-folder>`**.
- **Never** put `skills/`, `agents/`, `hooks/`, `commands/` inside `.claude-plugin/` — only `plugin.json` lives there.
- **Skills:** `skills/<name>/SKILL.md` with YAML frontmatter **`name`**, **`description`**, optional **`disable-model-invocation: true`** for `bootstrap` and `help`. The `name` field matches the folder and is required for **Codex** compatibility; `scripts/sync_cursor_skills.py` normalizes it when you regenerate Cursor copies.
- **Templates:** `mutter-claude/templates/CLAUDE.md`, `mutter-claude/templates/dot-mutter/`, and **`mutter-claude/templates/scripts/mutter.py`** (workspace CLI — bootstrap copies the latter to `<repo>/scripts/mutter.py` when missing). The dot-mutter tree mirrors canonical `.mutter/` when we change the scaffold; **bootstrap** skill tells the agent to copy `templates/dot-mutter` → `<repo>/.mutter/`. For **existing** `.mutter/`, consumers run **`python3 scripts/mutter.py bootstrap-sync`** (see workspace tools audit) to merge updated plugin files without wiping tasks or architecture. The template includes `memory/official-tech-docs-roadmap.md` (official doc URLs; check before web search).
- **Git hooks (optional):** `scripts/git-hooks/pre-commit` — enable repo-wide with `git config core.hooksPath scripts/git-hooks` (validates tasks + plans when `.mutter/` and `scripts/mutter.py` exist).
- **Dev:** `claude --plugin-dir ./mutter-claude`; `/reload-plugins` after edits.

## OpenAI Codex (`mutter-claude` + `.codex-plugin`)

Docs: [Plugins](https://developers.openai.com/codex/plugins), [Build plugins](https://developers.openai.com/codex/plugins/build). Cross-check [obra/superpowers](https://github.com/obra/superpowers) (`.codex-plugin/plugin.json` at plugin root).

- Manifest: **`mutter-claude/.codex-plugin/plugin.json`** only (per Codex path rules — no extra files inside `.codex-plugin/`). **`skills`** path is **`./skills/`** relative to `mutter-claude/`, i.e. the **same** Markdown skills as Claude.
- Skill files: each `skills/<id>/SKILL.md` uses frontmatter **`name`** + **`description`** (same bodies as Claude; headings may still say `/mutter:` for human parity).
- **Marketplaces:** Codex reads `$REPO_ROOT/.agents/plugins/marketplace.json` and **also** `$REPO_ROOT/.claude-plugin/marketplace.json`. This repo ships **`.agents/plugins/marketplace.json`** with a **local** entry pointing at **`./mutter-claude`**. Use `codex plugin marketplace add` / official flows to register Git-backed catalogs when distributing.
- **Kebab-case:** Codex docs ask for a stable plugin **`name`** in kebab-case; **`mutter`** is valid.

## Cursor (`mutter-cursor`)

Docs: [Plugins](https://cursor.com/docs/plugins), [Plugins reference](https://cursor.com/docs/reference/plugins.md).

- Manifest: `mutter-cursor/.cursor-plugin/plugin.json` — explicit `rules`, `skills`, `commands` paths replace default discovery for those types.
- **Multi-plugin repo catalog:** `.cursor-plugin/marketplace.json` at **repository root** lists the `mutter` plugin with `"source": "mutter-cursor"` so Teams/Enterprise can **import this GitHub repo** as a [team marketplace](https://cursor.com/docs/plugins) (Dashboard → Settings → Plugins → Team Marketplaces → Import). Public discovery still goes through [cursor.com/marketplace/publish](https://cursor.com/marketplace/publish) with the same repo URL.
- **Rules:** `mutter-cursor/rules/mutter.mdc` — same routing intent as `CLAUDE.md` and `docs/mutter-cursor-ide-routing.md`.
- **Skills:** same folder names as Claude; each `SKILL.md` has Cursor frontmatter **`name` + `description`** (generated by `scripts/sync_cursor_skills.py` from `mutter-claude/skills`).
- **Commands:** `mutter-cursor/commands/<skill>.md` with `name: mutter-<skill>` for palette/agent command discovery.
- **Local install:** symlink/copy to `~/.cursor/plugins/local/mutter` so `.cursor-plugin/plugin.json` is at plugin root → **Developer: Reload Window**.

## OpenCode (repo root)

Docs: [Plugins](https://opencode.ai/docs/plugins/). Reference implementation: [obra/superpowers](https://github.com/obra/superpowers) (`package.json` → `.opencode/plugins/*.js`, `config` hook pushes `skills.paths`, optional `experimental.chat.messages.transform`).

- **`package.json`**: `"name": "mutter"`, `"main": ".opencode/plugins/mutter.js"`, `"type": "module"`. Install with e.g. **`mutter@git+https://github.com/arnaudovproject/mutter.git`** in `opencode.json` → **`plugin`** array (see `.opencode/INSTALL.md`).
- **`.opencode/plugins/mutter.js`**: exports **`MutterPlugin`** — registers **`mutter-claude/skills`** on **`config.skills.paths`**; prepends bootstrap from **`mutter-claude/templates/CLAUDE.md`** on first user message (same pattern as Superpowers’ bootstrap injection).
- OpenCode **local** plugins under `.opencode/plugins/*.ts` are for **this repo’s** development only; the **distributable** story is the **git/npm** line above so consumers get the whole tree including `mutter-claude/skills`.

## Mutter workspace system (v1)

- **`.mutter/`** holds all durable project intelligence in **small shards** (architecture, roadmap, tasks, index JSON, logs, brainstore, etc.). See `.mutter/core/project.md`.
- **Entry files must stay tiny:** root `CLAUDE.md` and **`mutter-cursor/rules/mutter.mdc`** (plugin) / consumer **`.cursor/rules/mutter.mdc`** (optional copy after bootstrap) only route into `.mutter/` and state global safety/token rules.
- **Incremental scan** state: `.mutter/metadata/scan-state.json` (`hash_strategy`, `file_hashes`, `changed_files`).
- **Index shards:** `.mutter/index/*.json` — symbol/path/domain/test pointers, not full source.
- **Tasks:** `.mutter/tasks/{current,planned,blocked,completed}/` — one markdown task per file when possible.
- **Plans / reviews / diffs:** `.mutter/plans/`, `.mutter/reviews/`, `.mutter/diffs/` for durable audit trails.
- **State:** `.mutter/state/current.json` tracks active task/plan/workflow pointers and optional **`execution_progress`** (from `sync-task-progress`).

## Parity rules when changing behavior

1. Edit **`mutter-claude/skills/<x>/SKILL.md`** as the canonical skill body, then run **`python3 scripts/sync_cursor_skills.py`** to refresh **`mutter-cursor/skills`** (and normalize Claude frontmatter **`name`** / **`description`** / **`disable-model-invocation`**). CI with **`--check-cursor-sync`** diffs both **`mutter-claude/skills`** and **`mutter-cursor/skills`**.
2. Update **`.mutter/`** scaffold and re-run `cp -R .mutter mutter-claude/templates/dot-mutter` (or edit template copy consciously). When changing **`scripts/mutter.py`**, copy the same file to **`mutter-claude/templates/scripts/mutter.py`** so bootstrap stays in sync.
3. If routing text changes, update **both** `CLAUDE.md` (root + `mutter-claude/templates/CLAUDE.md`), **`mutter-cursor/rules/mutter.mdc`**, and **`docs/mutter-cursor-ide-routing.md`**. OpenCode bootstrap reads **`mutter-claude/templates/CLAUDE.md`** — keep it aligned with root intent.
4. After adding/removing a skill folder, add/remove **Cursor command** file and bump **`version`** in **Claude, Cursor, and Codex** manifests (`mutter-claude/.claude-plugin/plugin.json`, `mutter-cursor/.cursor-plugin/plugin.json`, `mutter-claude/.codex-plugin/plugin.json`, root **`package.json`**) when you ship a release.
5. After editing **Claude** skills, run **`python3 scripts/sync_cursor_skills.py`** (then refresh any hand-edited `mutter-cursor/commands/*.md` if names change).

## Token-efficiency non-negotiables

- No whole-repo dumps in rules or entry files.
- Use index shards + explicit file lists (`analyze`, `affected`, `tests`) before wide reads.
- Chat replies: short summaries + paths; large detail lives under `.mutter/`.
